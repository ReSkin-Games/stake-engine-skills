# Outputs

All output files land under `games/<game>/library/`. Missing subfolders are auto-created by `OutputFiles` (`math-sdk/src/config/output_filenames.py`).

## Generators

- `math-sdk/src/write_data/write_data.py` — book and lookup-table writers (`create_books`).
- `math-sdk/src/write_data/write_configs.py` — `config.json`, `config_fe.json`, `config_math.json` (`generate_configs`).
- `math-sdk/src/write_data/force.py` — `force_record_<betmode>.json`, aggregated `force.json`.

## `library/books/` and `library/books_compressed/`

Per-betmode files:

- `books/books_<betmode>.jsonl` — one JSON object per line, uncompressed. For local debugging only; do not upload.
- `books_compressed/books_<betmode>.jsonl.zst` — Zstandard-compressed JSON Lines. This is what the RGS consumes; only data from compressed books is returned from `/play`.

Each line:

```json
{
  "id": 1,
  "payoutMultiplier": 0.0,
  "events": [{...}, {...}],
  "criteria": "basegame",
  "baseGameWins": 0.0,
  "freeGameWins": 0.0
}
```

`events` is the exact array returned in the RGS `/play` response body.

`compression` is a flag on `create_books()`; setting it `False` produces only the uncompressed form.

## `library/lookup_tables/`

- `lookUpTable_<betmode>.csv` — `simulation_number, weight, payout_multiplier`. Initial weights are `1`.
- `lookUpTable_<betmode>_0.csv` — optimizer output; same shape, modified weights. **This is the file used at publish time.**
- `lookUpTableIdToCriteria_<betmode>.csv` — `simulation_number, criteria` (e.g. `freegame`, `wincap`, `0`).
- `lookUpTableSegmented_<betmode>.csv` — per-simulation basegame vs freegame win contribution.

The optimizer reads `lookUpTable_<betmode>.csv` and writes `lookUpTable_<betmode>_0.csv`. The two segmented/criteria files are for analysis and are not normally uploaded to the ACP.

## `library/forces/`

- `force_record_<betmode>.json` — entries written by `self.record()` calls; see `force-and-criteria.md`.
- `force.json` — aggregated union of unique search fields across all betmodes; manifest for prototyping.

## `library/configs/`

Three JSON files generated by `generate_configs(gamestate)`:

- `config.json` — backend; betmode info plus file hash values. The RGS uses hashes to verify ACP uploads.
- `config_fe.json` — frontend; symbols, padding, betmodes (cost/name/flags) needed to render UI.
- `config_math.json` — optimizer input; bet modes, RTP splits, optimization parameters from `src/config/optimization_paramaters.py`.

## `library/publish_files/`

Subset required for RGS upload — typically `books_<betmode>.jsonl.zst`, `lookUpTable_<betmode>_0.csv` per betmode, plus `index.json`. See `rgs-output-contract.md` for the publish-time contract.

## When each file is written

- `create_books()` populates `books/`, `books_compressed/`, `forces/`, `lookup_tables/{lookUpTable, lookUpTableIdToCriteria, lookUpTableSegmented}_<betmode>.csv`.
- Optimizer pass (`run_optimization: True`) adds `lookUpTable_<betmode>_0.csv`.
- `generate_configs(gamestate)` writes `configs/*.json`.
- Analysis pass (`run_analysis: True`) writes the PAR sheet using segmented + force data.
